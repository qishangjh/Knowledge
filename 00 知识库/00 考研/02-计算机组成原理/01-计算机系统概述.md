## 01-1 计算机发展历程

### 1. 计算机硬件的发展

#### 1. 计算机的四代变化

| 代数     | 时间区间     | 硬件元件更新                                     | 主要特点与时代背景                                                                       |
| :------- | :----------- | :----------------------------------------------- | :--------------------------------------------------------------------------------------- |
| **第一代** | 1946~1957 年 | 使用笨重的**电子管**作为核心逻辑元件。         | **电子管计算机**，体积庞大，能耗高，可靠性低。代表：ENIAC（世界第一台通用电子数字计算机）。  |
| **第二代** | 1958~1964 年 | 用小巧的**晶体管**代替了电子管。               | **晶体管计算机**，体积、功耗显著降低，可靠性提升；程序设计语言和操作系统的初步发展。    |
| **第三代** | 1965~1970 年 | 把多个晶体管集成到一块小小的**芯片**上。       | **中小规模集成电路（SSI/MSI）计算机**，性能大幅提升，成本下降，普及度增加；分时操作系统问世。 |
| **第四代** | 1972 至今    | 在一块芯片上集成成千上万甚至上亿个晶体管（**微处理器**）。 | **大规模和超大规模集成电路（LSI/VLSI）**，性能空前提高，微处理器广泛应用，推动个人计算机普及；网络、数据库、图形用户界面技术迅速发展。 |
| **第五代** | (研究阶段)   | 还在探索中，目标是让计算机更像人一样思考。       | **人工智能计算机**，研究重点在于并行处理、智能推理、模式识别等领域，旨在实现更高层次的智能计算。 |

> [!info] 什么是**分时操作系统**？
> 其核心思想是将CPU的运行时间划分为“时间片”，系统轮流将CPU的控制权分配给各在线用户的程序，每个程序只运行一个时间片。如此，使得多个用户能够通过各自的终端“同时”访问一台主机，并获得及时响应，显著提升了计算机系统的资源利用率和用户交互体验。

> [!link] **机器字长**
> **机器字长**：指计算机进行一次整数运算所能处理的二进制数据的位数。该指标直接决定了计算机能够处理的数据宽度、精度以及CPU的运算能力。例如，32位或64位处理器即指机器字长。

#### 2. 计算机元件的更新换代

计算机硬件的演进实质上是其核心电子元件的革新史。

*   **1946年**：世界第一台电子数字计算机 **ENIAC** 诞生，其核心逻辑元件为**电子管**，体积庞大且能耗高。
*   **1947年**：贝尔实验室的肖克利、巴丁、布拉顿发明了**晶体管**。这一发明具有划时代意义，晶体管相比电子管体积更小、能耗更低、可靠性更高。
*   **1955年**：肖克利在硅谷创建肖克利实验室股份有限公司，开启了硅谷的传奇。
*   **1957年**：八位科学家离开肖克利公司，共同创立了**仙童半导体公司**，标志着半导体产业黄金时代的开启。
*   **1959年**：仙童半导体公司发明了**集成电路**！此技术将晶体管、电阻、电容等元件集成到一块微小的硅片上，是计算机实现小型化的关键。
*   **1968年**：仙童公司的摩尔等人离开了仙童，创立了我们现在大名鼎鼎的 **Intel（英特尔）**！
*   **1969年**：仙童销售部负责人桑德斯也离开了仙童，创立了另一家巨头 **AMD（超微半导体）**！
    *   **注解：** 仙童公司在半导体产业发展中扮演了重要角色，培养了众多行业领军人物。

> [!link] **摩尔定律** (Moore's Law)
> 这是由Intel创始人之一戈登·摩尔提出的一条著名的经验性“定律”：**集成电路上可容纳的晶体管数目，大约每隔18个月便会增加一倍，整体性能也将提升一倍。** 该定律在数十年间深刻指导并推动了信息技术的发展，持续提升了计算机的性能。尽管当前其增长速度面临物理极限，但它仍是信息技术发展史上的重要里程碑。

*   **1970年**：仙童公司生产出第一个较大容量的**半导体存储器**。
    *   半导体存储器单芯片容量：从最早的1KB，一路飙升到4KB、16KB、64KB、256KB、1MB、4MB、16MB、64MB、256MB、1GB，直至更高容量。这直接决定了计算机的存储能力。

### 2. 计算机软件的发展

*   **早期阶段（20世纪50年代）**
    *   早期程序员直接采用**机器语言**（二进制代码）进行程序设计，效率极低，且开发难度大。
    *   很快，人们发明了**汇编语言**，通过使用助记符（如`ADD`、`MOV`）替代二进制代码，显著提高了编程效率。
*   **60年代**
    *   高级语言（如**FORTRAN**、**COBOL**、**ALGOL**）的出现，其更接近人类自然语言和数学表达，极大地降低了编程复杂度，使程序员得以脱离底层硬件细节。
    *   **操作系统**的雏形也开始形成，作为计算机的资源管理者，协调并控制硬件资源。
*   **70年代**
    *   **多道程序系统**、**分时系统**、**实时系统**等操作系统技术蓬勃发展，提高了计算机多任务处理能力和实时响应性能。
    *   **数据库系统**、**编译技术**（将高级语言翻译成机器语言）也逐步成熟。
*   **80年代以后**
    *   **面向对象编程**（如C++、Java）、**软件工程方法**（促使软件开发过程更加规范化、科学化）、**图形用户界面（GUI）**等兴起，显著提升了计算机的用户友好性和易用性。
    *   软件开发逐渐走向工程化、模块化，分工日益细化。
*   **现代软件发展趋势**
    *   云计算、大数据、人工智能、移动互联网等新兴技术持续涌现，共同推动软件产业的快速发展。
    *   现代软件系统日益复杂，因此，对软件系统的**可维护性**和**可扩展性**提出了更高要求。

## 01-2 计算机系统层次结构

计算机系统并非单一平面结构，其内部遵循严格的层次化组织原则。各层基于下层功能构建，并向上层提供服务，同时隐藏了底层实现细节。这种抽象模型即为**虚拟机模型**。

### 1. 计算机系统的组成

计算机系统是硬件与软件的有机融合。
*   **计算机系统**：由硬件系统和软件系统组成的整体，二者相辅相成，共同完成数据处理及各项复杂任务。
*   **硬件系统**：包括中央处理器（CPU）、主存储器（主存）、输入/输出设备、总线等。
*   **软件系统**：包括系统软件（操作系统、编译器、汇编程序、各类实用程序）和应用软件（用户日常使用的各类应用程序）。
*   **软硬件协同工作**：软硬件系统紧密协作，共同构成完整的计算机系统，以实现预设功能。

> [!tip] 硬件与软件哪个更重要？
> 二者相辅相成，缺一不可。硬件是基础，提供软件运行的物理平台；软件是核心，指挥硬件执行特定功能。没有软件，硬件即为无用；没有硬件，软件亦无法运行。因此，二者在不同层面发挥关键作用，不可简单判定重要性高低。通常认为，计算机系统的整体性能取决于软硬件协同优化的综合效果。

### 2. 计算机硬件

#### 1. 冯·诺依曼机基本思想
冯·诺依曼（Von Neumann）提出的 **“存储程序”思想**是现代计算机体系结构的基石。
*   **核心思想**： **存储程序**。其核心理念是将指令和数据均以**二进制**形式**统一存储于存储器（内存）中**，计算机依此顺序自动取出指令并执行，从而实现程序的自动化运行。 #2019年

| 特征         | 指令 (Instruction)                                         | 数据 (Data)                                              |
| :----------- | :--------------------------------------------------------- | :------------------------------------------------------- |
| **用途**     | 指明CPU执行的操作，是操作码和操作数的集合。              | CPU进行运算和处理的对象，是原始信息或计算结果。        |
| **构成**     | 操作码（定义操作）和操作数（指明操作对象）。             | 表示数值、字符等各种形式的原始或加工信息。              |
| **解释方式** | 由CPU的**控制单元**译码并控制执行流程。                  | 由CPU的**运算器**执行操作，或由**控制单元**作为地址进行寻址。 |
| **存储位置** | 程序区（代码段）。                                         | 数据区、堆区、栈区等。                                   |
| **寄存器**   | 在取指阶段送入**指令寄存器 (IR)**。                       | 在执行阶段送入**通用寄存器 (GPR)** 或其他专用寄存器，或作为地址供MAR使用。 |
| **生命周期** | 在程序执行过程中被周期性地取指、译码、执行。             | 在程序执行过程中被动态创建、修改、读取及销毁。           |
| **修改**     | **通常不允许在程序运行中被修改**（代码段通常设置为只读）。 | **可以被程序根据需求进行修改**（如变量赋值）。         |
| **获取阶段** | 在**取指周期 (Fetch Cycle)** 从存储器中获取。             | 在**执行周期 (Execute Cycle)** 中根据指令要求从存储器中访问。 |

| **概念**             | **定义**                                           | **特征**                                                     |
| :------------------- | :------------------------------------------------- | :----------------------------------------------------------- |
| **冯·诺依曼架构**   | 指令与数据共享同一存储空间的计算机体系结构         | 统一编址、指令与数据统一存储；CPU通过**时序**来区分指令/数据 |
| **指令周期**         | CPU 执行一条指令的完整过程                         | 包含取指、译码、执行、访存、写回等阶段；每个阶段有特定任务和时序 |
| **哈佛架构**         | 指令与数据物理上分离存储的计算机体系结构           | 指令存储器和数据存储器独立；可并行访问指令和数据，提高效率 |

>[!key] **【考点】冯·诺依曼计算机中指令和数据区分依据 **
>在冯·诺依曼体系结构中，CPU区分从统一存储器中获取的二进制信息是指令还是数据的根本依据在于其所处的**指令周期的不同阶段**：在取指阶段，CPU默认获取的是指令；在执行阶段（或访存阶段），CPU根据指令要求访问存储器，此时获取的默认是数据。CPU无法从内容的格式或物理位置上直接区分两者。

>[!采用二进制编码表示，其主要原因]
> #2018年
> Ⅰ. 二进制的运算规则简单
> Ⅱ. 制造两个稳态的物理器件较容易
> Ⅲ. 便于用逻辑门电路实现算术运算

#### 2. 计算机的功能部件
![[早期冯诺依曼机架构图.png|早期冯诺依曼机架构图]]
![[现代计算机组织结构.png|现代计算机组织结构]]

>[!工作过程]
>“存储程序 -> CPU译码执行 -> I/O交互反馈”的循环

*   **计算机五大功能部件详述**：
    *   **CPU (Central Processing Unit)**：中央处理器，是计算机的核心部件。它负责解释和执行程序指令，进行各种计算和控制。其自身包含两大核心部件：
        *   **运算器 (ALU - Arithmetic Logic Unit)**：是CPU中执行算术运算（加减乘除）和逻辑运算（与或非，比较大小）的核心部件。
        *   **控制器 (CU - Control Unit)**：是CPU中负责解释指令、发出控制信号，协调计算机各个部件工作的关键部件，确保指令按部就班地执行。
    *   **存储器 (Memory)**：是计算机用于存放程序和数据的设备。它分为：
        *   **主存储器（主存/内存）**：CPU能直接访问的存储器，速度快，但容量相对较小，断电数据会丢失。(类比人类短期记忆)。
        *   **辅助存储器（外存）**：如硬盘、U盘等，容量大，但速度慢，断电数据不丢失。(类比人类长期记忆或外部资料库)。
    *   **输入设备 (Input Device)**：是计算机用于接收外部信息并将其转换为机器可处理数据的设备。例如：键盘、鼠标、扫描仪、麦克风。
    *   **输出设备 (Output Device)**：是计算机用于将处理结果展示给用户或控制外部设备的装置。例如：显示器、打印机、音响。

*   **计算机硬件的主要技术指标**：
    *   **主频 (Clock Rate / Clock Frequency)**：CPU每秒钟的时钟周期数，单位为Hz（如GHz）。主频越高，CPU理论上完成指令的速度越快，但它并非影响指令执行速度的唯一因素。
    *   **字长 (Word Length)**：指CPU在**一次操作中能处理的二进制数据的位数**。该指标决定了CPU单次操作的数据位宽，影响数据通路宽度、CPU内部总线宽度、运算器位数及通用寄存器宽度。
        *   **数据通路**：指CPU内部数据流动所经过的路径及其相关部件，主要包括进行数据运算、存储和传送的部件。为确保高效工作，这些部件的位宽通常保持一致。
    *   **存储容量**：主存能存放的二进制信息总量，通常用MB/GB/TB表示。容量越大，计算机能同时运行的程序和处理的数据就越多。
    *   **存取速度**：存储器读取或写入数据所需的时间，单位通常为ns（纳秒）。速度越快，CPU等待数据的时间就越短。
    *   **吞吐量（MIPS/FLOPS）**：系统在单位时间内处理任务的数量。MIPS（每秒百万条指令）和FLOPS（每秒浮点运算次数）是衡量CPU性能的指标，其衡量方式包括MIPS和FLOPS等指标。
    *   **加速比 (Speedup)**：改进后的系统与原系统的性能比值。(改进后系统执行同一任务所需时间与原系统所需时间的比值)。
    *   **并行度 (Parallelism)**：指计算机系统同时处理多个任务或指令的能力。体现为多核 CPU、超线程技术、流水线技术、多线程编程等。目标是提升系统的多任务并发处理能力。

> [!tip] **关于"字长"**
> *   **机器字长 (Word Length)**：通常指**CPU的字长**，即CPU一次能处理的数据位数（如32位或64位）。它影响CPU的计算能力和数据处理效率。
> *   **存储字长 (Memory Word Length)**：指存储器中一个**存储单元**（一个地址对应的数据）的位数。在现代计算机中，存储器通常采用**字节（8位）编址**，故最小的存储字长为8位。然而，CPU在访问时，会依据自身的机器字长（如32位或64位）读取相应字节数。
> *   **指令字长 (Instruction Word Length)**：指一条指令所占的**位数**。可以是固定长度（所有指令长度一致），也可以是可变长度（不同指令长度不同）。
> *   **地址总线宽度 (Address Bus Width)**：指地址总线的**位数**，它决定了CPU能直接访问的**最大内存空间**。例如，32位地址总线能访问 $2^{32}$ 个地址（4GB），64位地址总线能访问 $2^{64}$ 个地址。值得注意的是，机器字长与地址总线宽度是相关但不等同的概念。

### 3. 计算机软件

#### 1. 系统软件和应用软件

*   **系统软件**：
    *   **操作系统（OS）**：作为计算机的管理者，负责调度和管理硬件资源，并为用户提供友好的操作环境。例如Windows、macOS、Linux、Android、iOS。
    *   **编译程序（Compiler）**：将高级语言（如C/Java等高级语言源代码）翻译成机器语言（计算机可执行的二进制代码）。
    *   **汇编程序（Assembler）**：将汇编语言（助记符形式的汇编语言代码）翻译成机器语言。
    *   **实用程序**：辅助计算机管理和维护的工具软件，如文件管理、磁盘管理、杀毒软件等。
*   **应用软件**：
    *   直接面向用户，完成特定任务的软件。例如：办公软件（Word, Excel）、图像处理软件（Photoshop）、影音播放软件、游戏软件、浏览器等。它们在系统软件之上运行，实现特定应用功能。
*   **支撑软件**：
    *   也称“中间件”或“开发工具”，辅助软件开发人员进行软件开发、调试和维护的工具或平台。例如：软件开发工具（IDE，如Visual Studio Code）、调试工具、版本管理工具（Git）等。

#### 2. 三个级别的语言

计算机编程语言，从高级抽象到机器可执行，可分为三个级别：

| 语言类型                | 特点                                                                 | 举例                     | 转换方式与联系                                                                   |
| :------------------ | :----------------------------------------------------------------- | :--------------------- | :------------------------------------------------------------------------------- |
| **高级语言**            | **面向用户**，接近人类自然语言和数学表达，具备**高可读性和编写效率**，但需经翻译后方可由机器执行。              | C、Java、Python、Pascal 等 | 通过**编译器（Compiler）**或**解释器（Interpreter）**翻译为汇编语言或机器语言。 |
| **汇编语言**            | **面向机器**，采用助记符（如`ADD`、`MOV`）表示机器指令，可读性优于机器语言，与硬件结构紧密相关，执行**效率较高**。 | `ADD AX, BX`（加法指令）     | 通过**汇编器（Assembler）**转换为机器语言。与机器语言**一一对应**。           |
| **机器语言**<br>#2015年  | **面向机器**，完全由**二进制代码**组成，**CPU可直接识别执行**，执行效率最高，但可读性与编写难度极高。        | `00110011`（机器码）        | 汇编语言的最终执行形式，也是所有高级语言程序最终要转换为的形式。                      |

#### 3. 软件和硬件的逻辑功能等价性
*   **逻辑功能等价性定义**：
    *   指**相同的功能可以通过硬件电路实现，亦可通过软件编程实现，且最终逻辑效果一致。**
*   **产生机制**：
    *   根源在于计算机底层均为二进制逻辑运算，无论是硬件门电路还是软件逻辑判断，其本质均是布尔代数的实现。
*   **意义与权衡**：
    *   **灵活性**：软件实现的功能修改灵活，成本低；硬件实现则修改困难，通常需重新设计电路。
    *   **效率与成本的权衡**：
        *   **硬件实现**通常具有更高的执行速度，源于物理电路的直接执行。但其设计成本高、周期长，且一旦制造完成，修改难度大。
        *   **软件实现**执行速度相对较低，因需经多层翻译和解释。但开发成本低、周期短，且修改灵活。
    *   因此，在实际计算机系统设计中，工程师会基于对性能、成本、灵活性的综合考量，决定功能的硬件或软件实现方式。

### 4. 计算机系统的层次结构

计算机系统呈现出多层次的结构特点。每一层均基于下层功能构建，并向上层提供服务，同时封装了底层的实现细节。这种抽象模型即为**虚拟机模型**。

*   **第6层：应用语言层**：
    *   最顶层，直接面向最终用户，如各类应用软件。
*   **第5层：高级语言层**：
    *   由C、Java、Python等高级语言编写的程序。该层程序需经**编译器或解释器**转换为下一层的形式。
*   **第4层：汇编语言层**：
    *   由汇编语言程序组成。该层程序需经**汇编器**转换为下一层的形式。
*   **第3层：操作系统层**：
    *   提供系统调用接口和资源管理服务。应用程序和高级语言程序均通过操作系统间接控制硬件资源。
*   **第2层：指令集体系结构（ISA）层**：
    *   是软件与硬件的**分界接口**。它定义了计算机可执行的指令集合、指令格式、寻址方式、寄存器组织、数据类型等，是程序员可见的硬件功能抽象。从程序员视角看，这是机器能够直接理解并执行的语言。
*   **第1层：微体系结构层**：
    *   CPU内部的具体实现机制，例如流水线技术、缓存设计、分支预测等。它实现了ISA层定义的指令集，但这些细节对ISA层而言是透明的。
*   **第0层：数字逻辑层**：
    *   最底层，由门电路、触发器等基本的电子元件和逻辑电路组成，是计算机硬件的物理基础。

> [!tip]
> 各层通过定义的**接口**与相邻层通信，上层服务使用者无需了解下层的具体实现细节。这种分层设计提升了系统的模块性、可维护性与可管理性。

### 5. 计算机系统的工作原理

#### 1. "存储程序"工作方式（冯·诺依曼）
*   **核心**：程序和数据都以二进制形式**存储在内存中**。
*   **过程**：
    *   程序由一系列按逻辑顺序排列的指令组成。
    *   **控制器**根据程序计数器（PC）中存储的指令地址，自动从主存储器中取出指令。
    *   随后对指令内容进行分析，依据指令操作码控制**运算器**、**存储器**等部件执行相应操作。
    *   一条指令执行完毕后，控制器自动准备获取下一条指令。
*   **意义**：这种“存储程序，顺序执行”的工作模式是现代计算机实现**自动化、高效运行**的基石，极大提升了程序的执行效率并避免了人工干预的繁琐。

#### 2. 从源程序到可执行文件
#2022年  #2016年

| 阶段   | 工具/处理器      | 文件类型       | 主要功能                               |
| :----- | :--------------- | :------------- | :------------------------------------- |
| 预处理 | 预处理器CPP      | 预处理文件.i   | 处理#开头命令（如`#include`, `#define`） |
| 编译   | 编译器ccl        | 汇编源程序.s   | 生成汇编语言代码                       |
| 汇编   | 汇编器as         | 目标文件.o     | 翻译成**机器语言指令**                 |
| 连接   | 连接器ld         | 可执行文件     | 合并目标文件和标准库函数               |

1.  **预处理 (Preprocessing)**：
    *   源程序（`.c` 或 `.cpp` 文件）首先经过预处理器处理。
    *   预处理器处理`#include`（将头文件内容插入）、`#define`（进行宏替换）等指令。
    *   生成一个**预处理文件**（通常为`.i`文件），其中已包含了所有引入的代码，且宏已完成展开。
2.  **编译 (Compiling)**：
    *   预处理文件由**编译器**接收。
    *   编译器将高级语言代码翻译成**汇编语言代码**（`.s` 文件）。此过程涉及词法分析、语法分析、语义分析、中间代码生成及代码优化等复杂步骤。
3.  **汇编 (Assembling)**：
    *   汇编语言代码由**汇编器**接收。
    *   汇编器将汇编语言代码翻译成**机器语言代码**，生成**目标文件**（`.o` 或 `.obj` 文件）。目标文件已是二进制形式，但其不完整性在于可能存在对外部库函数的引用。
4.  **链接 (Linking)**：
    *   链接器接收多个目标文件（由不同源文件编译生成）以及程序所引用的各类**库文件**（例如包含`printf`函数的标准库）。
    *   链接器解析并确定模块间的引用关系（例如确定`printf`函数在内存中的实际地址）。
    *   最终生成一个完整且可在操作系统环境下直接运行的**可执行文件**（如Windows下的`.exe`文件或Linux下的无后缀二进制文件）。

> [!tip] 完整转换过程
> 高级语言文本 -> 预处理文本 -> 汇编文本 -> 二进制目标文件 -> 二进制可执行文件

#### 3. 执行过程的描述（CPU的“取指-分析-执行-更新”循环）

当可执行文件被加载到内存并开始运行时，CPU便进入一个周而复始的“取指-执行”循环：

1.  **取指令 (Fetch Instruction)**：
    *   **程序计数器（PC）** 存储着下一条待执行指令的地址。
    *   **控制器**根据PC中的地址，向**存储器**发出读取请求，将指令从存储器中取出，存入**指令寄存器（IR）**。
    *   同时，程序计数器（PC）的值自动更新，指向下一条指令的地址。
2.  **分析指令 (Decode Instruction)**：
    *   **控制器**对指令寄存器（IR）中保存的指令进行分析，识别出指令的**操作码**（表示要执行的操作，如加法、数据传送）和**操作数地址**（表示操作数的位置）。
>[!tip] **【真题关联】条件转移指令与标志寄存器 **
    > 在**分析/执行指令**过程中，CPU执行算术逻辑运算后，会更新**标志寄存器**（如CF、ZF、SF、OF）。**条件转移指令**（如`bgt`）会依据这些标志位的状态决定程序执行流。例如，无符号整数比较`A > B`时，通常判断条件为`CF = 0`且`ZF = 0`（无借位且结果不为零）。
    > *(关于标志寄存器的详细内容，请参考【第二章 数据的表示和运算】和【第四章 指令系统】。)*
3.  **执行指令 (Execute Instruction)**：
    *   依据操作码，**控制器**发出相应的控制信号，驱动**运算器**、**存储器**、**I/O设备**等部件完成具体操作。
    *   例如，对于加法指令，运算器将从存储器中获取操作数，执行加法运算，并将结果存回存储器或寄存器。
    *   若遇条件或无条件跳转指令，控制器将根据指令中指明的跳转目标地址修改PC值，实现程序的控制流转移。
4.  **更新程序计数器（Update PC）**：
    *   通常情况下，PC在取指令后会自动指向下一条连续的指令。
    *   若执行了跳转指令（例如`if`语句、`for`循环对应的机器指令），控制器会根据指令的跳转目标地址修改PC的值，使程序跳转到新的地址继续执行。
    *   >[!danger] **【考点】中断与指令执行 **
    > **在开中断状态下，CPU通常会在当前指令执行结束时检查是否存在外部中断请求。** 若有，则CPU会转而处理中断服务程序，从而打断当前程序的连续执行。**但是，CPU并非在每来一个时钟脉冲信号时就开始执行一条新的完整指令**，一条指令的执行往往需要多个时钟周期。
*   **总结：** 程序的执行即是这样一个循环往复的过程，CPU不断地从存储器中取出指令、分析指令、执行指令，直至程序运行结束。这即是“存储程序，顺序执行”原理的精髓。

## 01-3 计算机性能指标

### 一、计算机的主要性能指标

这些参数直接反映了计算机系统的工作能力和效率，是评估其性能的核心标准。

*   **1.1 吞吐量 (Throughput)**
    *   **定义：** 单位时间内系统能处理的任务数。(类比快递分拣中心单位时间内的包裹处理量)。
    *   **目标：** 越高越好！
*   **1.2 响应时间 (Response Time)**
    *   **定义：** 用户提交一个任务到系统给出响应的时间。(用户发起请求至系统给出结果的时间)。
    *   **目标：** 越短越好！
*   **1.3 CPU 时钟周期 (Clock Cycle)** #2025年
    *   **定义：** CPU 中最小的时间单位，是 CPU 完成一个基本动作所需的时间。
    *   **关系：** **主频的倒数**。例如，主频 1GHz = $10^9$ Hz，则时钟周期为 $1/(10^9)$ 秒 = 1 纳秒 (ns)。
    *   >[!tip] **【考点】时钟周期确定依据 **
    > 时钟周期的长度必须以**相邻状态单元间组合逻辑电路的最大延迟**为基准确定，以确保信号在下一个时钟沿到来前稳定。
*   **1.4 主频 (Clock Rate / Clock Frequency)**
    *   **定义：** CPU 每秒钟的时钟周期数，单位为 Hz (如 GHz)。可理解为CPU内部时钟的脉冲频率。 #2017年
    *   **影响：** 主频越高，CPU 理论上完成指令的速度越快。但它只是影响指令执行速度的一个因素，不是唯一因素。
    *   >[!info] **【考点】时钟脉冲信号 **
    > 时钟脉冲信号由机器脉冲源发出，经整形和分频后形成。其倒数为主频。处理器**并非**总是在每来一个时钟脉冲信号时就开始执行一条新的**完整指令**，一条指令的完成通常需要多个时钟周期。
*   **1.5 CPI (Cycles Per Instruction)**
    *   **定义：** 执行一条指令平均需要的时钟周期数。不同的指令类型，需要的周期数可能不同，CPI 是平均值。 #2014年 #2022年 #2023年
    *   **影响：** CPI 越小，说明 CPU 执行每条指令的效率越高，越好
>[!danger] **【考点】CPI与Cache缺失率 **
    > 程序的**CPI与Cache缺失率密切相关**。Cache缺失会导致CPU转而访问速度较慢的主存，引入额外等待时间，从而**增加指令的执行周期数，进而提高CPI**。

>[!tip] **【考点】CPI与CPU设计 **
    >*   **不同类型指令的CPI可能不一样**（如简单算术指令与复杂访存指令）。
    >*   **单周期CPU**的时钟周期以**最耗时指令**所用的时间为准。
    >*   **流水线CPU**的时钟周期以**最长流水段**所用时间为准。
    > *(详细内容将在【第三章 存储器层次结构】和【第五章 中央处理器(CPU)】中深入探讨。)*
*   **1.6 CPU 执行时间 (CPU Execution Time)**
    *   **定义：** CPU 真正用于执行程序指令所花费的时间。
    *   **计算公式 (非常重要！)：**
        $$ \text{CPU 执行时间} = \text{总指令数} \times \text{CPI} \times \text{时钟周期} $$
        或
        $$ \text{CPU 执行时间} = \frac{\text{总指令数} \times \text{CPI}}{\text{主频}} $$
    *   **理解：** 这个指标是衡量 CPU 性能最直接、最准确的指标之一，它综合了指令数量、指令执行效率和 CPU 速度。
*   **1.7 MIPS (Million Instructions Per Second)**
    *   **定义：** **每秒执行百万条指令数**。衡量CPU执行**整数指令**的平均速率。 #2013年
    *   **计算公式：**
        $$ \text{MIPS} = \frac{\text{指令数}}{\text{执行时间} \times 10^6} $$
        或
        $$ \text{MIPS} = \frac{\text{主频}}{\text{CPI} \times 10^6} $$
    *   **影响：** MIPS 越高，说明 CPU 处理指令的能力越强，越好！
*   **1.8 FLOPS (Floating Point Operations Per Second)**
    *   **定义：** **每秒浮点运算次数**。专门衡量计算机执行**浮点数（带小数位）运算**的能力。
    *   **重要性：** 在科学计算、人工智能 (AI)、图像处理、大数据分析等需要大量浮点运算的领域，这个指标非常重要。
    *   **扩展 (近年新增考点):** MFLOPS, GFLOPS, TFLOPS, PFLOPS, EFLOPS, ZFLOPS 分别代表 $10^6, 10^9, 10^{12}, 10^{15}, 10^{18}, 10^{21}$ 次浮点运算/秒。 #2021年
*   **1.9 加速比 (Speedup)**
    *   **定义：** 改进后的系统与原系统的性能比值。(改进后系统执行同一任务所需时间与原系统所需时间的比值)。
    *   **目标：** 加速比越大，说明改进效果越显著。
*   **1.10 并行度 (Parallelism)**
    *   **定义：** 指计算机系统同时处理多个任务或指令的能力。
    *   **体现：** 多核 CPU、超线程技术、流水线技术、多线程编程等都是为了提高并行度。
    *   **目标：** 提升系统的多任务并发处理能力。

### 二、专业术语辨析

在计算机组成原理领域，存在部分术语易混淆但内涵迥异。精确理解这些概念的层次与界限，是构建清晰知识体系的基础。

*   **2.1 指令 (Instruction)**
    *   **定义：** 计算机执行一个基本操作的命令，是计算机的最小功能单位。(例如算术运算、数据传送、控制流转移等操作命令)。
*   **2.2 程序 (Program)**
    *   **定义：** 由多条**指令**按照一定的逻辑顺序组成的集合，用于完成特定的、复杂的任务。(通常由指令序列、数据及控制流逻辑构成)。
*   **2.3 指令集体系结构 (ISA, Instruction Set Architecture) #近年新增**
    *   **定义：** ISA是**软件与硬件之间的接口规范**。它定义了计算机可执行的指令集合、指令格式、寻址方式、寄存器组织、数据类型等，是程序员可见的硬件功能抽象。不同的CPU系列（如x86、ARM）拥有各自的ISA。
*   **2.4 体系结构 (Architecture)**
    *   **定义：** 更广义的说法，通常指指令集设计、数据类型、寻址方式等**对程序员可见的属性**。在很多语境下与ISA互换使用，侧重于从程序员视角描述计算机的功能特性。
*   **2.5 微体系结构 (Microarchitecture)**
    *   **定义：** 指指令集的**具体实现方式**，是硬件工程师的设计范畴。如指令流水线、缓存结构、分支预测、ALU具体电路实现等。这些细节对程序员通常是**透明的**。
*   **2.6 计算机组织 (Computer Organization)**
    *   **定义：** 指计算机硬件各部件的**互联方式和资源组织方式**，如主存、寄存器、总线结构、I/O 系统等。关注计算机各功能部件的互联方式、控制逻辑以及资源组织结构，如CPU与主存的通信机制。
*   **2.7 计算机实现 (Computer Implementation)**
    *   **定义：** 最底层的物理实现，如具体的芯片制造工艺（多少纳米）、电路设计、布线技术等。指计算机硬件在物理层面的具体实现，包括芯片制造工艺、电路布线技术等工程细节。

### 三、计算机组成核心概念表格辨析

以下为计算机组成原理中基础且关键的术语，精确理解其含义对于构建清晰知识体系至关重要。

| 概念           | 含义                                                            |
| :------------- | :-------------------------------------------------------------- |
| **主机**       | **CPU 与主存** 的合称。(CPU与主存的集合)。                    |
| **CPU**        | **中央处理器**，计算机的"司令部"。(负责指令执行、算术逻辑运算及控制操作)。 |
| **主存**       | **内存**，存放运行中的程序和数据，CPU 能直接访问，断电信息丢失。(CPU可直接访问的高速存储器，断电数据丢失)。 |
| **存储单元**   | 存储器中可以**独立寻址**的最小单位，通常存储一个**字**或一个**字节**。(具有独立地址，可存储一个字或一个字节的数据区域)。 |
| **存储元件**   | 构成存储单元的**物理元件**，如电容 (DRAM)、触发器 (SRAM) 等。(构成存储单元的物理器件，如电容、触发器)。 |
| **存储基元**   | 存储**一位**二进制信息的最小单位。(存储一位二进制信息的最小物理单元)。 |
| **存储元**     | 同上，也是指存储一位二进制信息的最小单位。(存储一位二进制信息的最小物理单元)。 |
| **存储字**     | 一个存储单元中**存储的一组二进制数**。(一个存储单元所存储的二进制数据集合)。 |
| **存储字长**   | 一个存储字（即一个存储单元）所包含的**位数**。(一个存储字所包含的位数，通常为8位的整数倍)。 |
| **存储容量**   | 存储器中可存储的**总位数或总字节数**。(存储器可存储的总位数或总字节数)。 |
| **机器字长**   | **CPU 一次处理的二进制位数**，也称**CPU 字长**。(CPU单次处理二进制数据的位数，亦称CPU字长)。 |
| **指令字长**   | 一条指令所占的**位数**。(一条指令所占用的位数)。             |
